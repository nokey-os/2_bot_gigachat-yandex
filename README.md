# Telegram Unified AI Bot

Объединённый Telegram-бот, который может отвечать на вопросы через **GigaChat (Сбер)** или **YandexGPT (Яндекс)** — пользователь выбирает провайдера перед отправкой вопроса.

## Возможности

- Два AI-провайдера в одном боте
- Переключение между ними командами `/gigachat` и `/yandex`
- Inline-клавиатура для выбора провайдера
- Поддержка прокси (SOCKS5) и DNS-патча для обхода блокировок Telegram
- Любой из провайдеров можно отключить, просто убрав его ключи из `.env`

## Файлы проекта

| Файл | Назначение |
|------|-----------|
| `main.py` | Точка входа: настройка прокси/DNS, запуск polling |
| `bot.py` | Обработчики команд и сообщений, логика выбора провайдера |
| `config.py` | Загрузка конфигурации Telegram + провайдеров из `.env` |
| `gigachat_ai.py` | Асинхронный клиент для GigaChat |
| `yandex_ai.py` | Асинхронный клиент для YandexGPT |
| `.env` | Переменные окружения (токены, ключи) |

## Установка

```bash
# Клонировать / перейти в папку проекта
cd github.com/nokey-os/2_bot_gigachat-yandex/dit/main/README.md)

# Установить зависимости
pip install -r requirements.txt
```

## Настройка

Скопируйте `.env.example` в `.env` и заполните:

```ini
# Обязательно: токен Telegram-бота (от @BotFather)
TELEGRAM_BOT_TOKEN=your_token

# === GigaChat (достаточно AUTHORIZATION_KEY) ===
AUTHORIZATION_KEY=your_gigachat_key

# === YandexGPT (нужны оба ключа) ===
YC_API_KEY=your_yandex_cloud_api_key
YC_FOLDER_ID=your_yandex_cloud_folder_id
```
## Запуск

```bash
python main.py
```

## Использование в Telegram

| Команда | Действие |
|---------|----------|
| `/start` | Приветствие + клавиатура выбора провайдера |
| `/gigachat` | Переключиться на GigaChat |
| `/yandex` | Переключиться на YandexGPT |
| Любой текст | Отправить вопрос текущему провайдеру |

## Как это работает

1. Пользователь пишет сообщение боту
2. Бот определяет, какой провайдер выбран для этого пользователя (по `user_id`)
3. Отправляет сообщение в соответствующий AI-API
4. Возвращает ответ в Telegram

Выбор провайдера хранится в оперативной памяти (словарь `_user_providers`) и сбрасывается при перезапуске бота.
